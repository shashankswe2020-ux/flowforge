One prompt in. A shipped repo out.

swe-forge is a multi-agent LangGraph framework for autonomous software development. It runs your idea through a 10-node pipeline — clarification, spec, plan, parallel build, quality gates, triage, ship — and commits the artifacts to a brand-new GitHub repo with issues filed, a pull request opened, and a tagged release pushed.

Features

Everything an autonomous engineer needs.

swe-forge is not a one-shot code generator. It is a graph of specialized agents that plan, build, review, and ship — with the receipts to prove it.

🧭

Clarify before code

A clarification node turns a fuzzy prompt into a concrete brief — scope, constraints, acceptance criteria — before a single file is written.

📐

Spec-driven by default

Every run produces a markdown spec in docs/specs/ that the downstream agents must satisfy. No spec, no code.

🧩

Parallel builders

The planner fans out independent tasks to builder agents that work concurrently, each with its own slice of the spec and its own tests.

🛡️

5-axis review gate

Correctness, readability, architecture, security, performance. A reviewer agent blocks the ship until the bar is met.

🐛

Issues that triage themselves

Findings from review and security audits are filed as real GitHub issues, labelled and linked to the offending lines.

🚀

Ships a tagged release

On green, swe-forge opens a PR on a feature branch and pushes a semver tag. You review the diff, not a chat transcript.

How it works

From one line to a release in four moves.

  1. 01

    You write one line

    swe-forge run "build tic-tac-toe web app" — that is the entire specification you need to provide. The clarification agent fills in the rest by asking targeted questions when needed.

  2. 02

    The graph plans the work

    The spec node writes a markdown specification. The planner breaks it into discrete, dependency-ordered tasks and fans them out to parallel builder agents.

  3. 03

    Quality gates run in parallel

    Tests, lint, type-check, code review, and security audit all run as graph nodes. Failures route back to a triage agent that files issues and dispatches fix-up work.

  4. 04

    It ships to GitHub

    A new repo is created via gh, the feature branch is pushed, a pull request is opened, and on green a tagged release lands on main.

The pipeline

Ten nodes. One graph. Zero hand-holding.

Each node is a focused agent with a contract — a typed input, a typed output, and a quality bar it must clear before the graph advances.

Quick start

Three commands. That is the whole tutorial.

  1. Install

    Grab the package from PyPI. The distribution is swe-forge; the Python module is flowforge.

  2. Configure

    Pick a provider — GitHub Copilot, OpenAI, or Claude — in a one-time interactive setup. Credentials stay on your machine.

  3. Run

    Hand it a prompt. swe-forge creates a new private GitHub repo, runs the pipeline, and pushes the result.

bash 
# 1. install from PyPI
pip install swe-forge

# 2. configure provider/model (one-time, interactive)
swe-forge setup

# 3. run a prompt — auto-creates a private GitHub repo
swe-forge run "build tic-tac-toe web app"

# common flags
swe-forge run "<prompt>" --repo my-name      # use/create a specific repo
swe-forge run "<prompt>" --skip-github       # local-only, no remote
swe-forge run "<prompt>" --no-studio         # skip LangGraph Studio

Requires Python ≥ 3.12 and the gh CLI authenticated. Generated projects land in ~/flowforge-workspace/<slug>/ and are pushed to https://github.com/<you>/<slug>. The swe-forge source repo is never written to.

Stop prompting. Start shipping.

swe-forge turns a sentence into a repository — with a spec, a plan, tests, a review, issues, a PR, and a tag. Try it on your next side-project tonight.
